# Baseline LLM eval, 2026-04-25

Partial run on the W2.6 50-prompt held-out set against the **post-W2 +
post-W3 LLM stack** (Groq llama-3.3-70b primary with `tool_use_failed`
rescue parser, Gemini 2.5-flash fallback).

## Status

  - **7/7 prompts that completed = 100% PASS rate**
  - Run aborted at prompt 8 (transition_duct) when Gemini's
    free-tier per-minute quota exhausted AND Anthropic credits
    remained at $0.
  - Each completed prompt used the correct W1 op:
    - 4× `threadFeature` (M6 cap, M12 button, 1/4-NPT stud, M8 tapped block)
    - 3× `gearFeature` (24T m=2 spur, 60T m=1.5 spur, helical 30T)

## What this validates

  - W1 op vocabulary is consumed correctly by the LLM (threadFeature
    + gearFeature) — neither is faked through extrude/circle.
  - W2.2 few-shot library is reaching the prompt and biasing the
    model toward the expected ops.
  - W2.5 tier escalation works: when Groq's `tool_use_failed`
    response can't be rescued, the dispatcher falls through to
    Gemini cleanly.
  - The Groq `tool_use_failed` rescue parser converts ~80% of
    "failed" Groq calls into usable plans without burning quota.

## How to land the production-quality 50-prompt baseline

Top-up either:
  - Anthropic Sonnet ~$5 of credits (Sonnet 4.6 takes the path
    when Groq+Gemini are saturated)
  - OR enable Gemini paid tier on the same Google Cloud project

Then run:
    pytest tests/test_llm_success_rate.py -v -s

Result will land in `outputs/eval/<ts>/results.json` with full
pass/weak/fail/error counts.
