## Bug fixed during testing

**`test_llm_tiers.py` — 6 failures from Groq addition**
The W2/eval-fix work added `_try_groq` to the `call_llm` chain
(balanced/fast first, premium third) but the existing tier-order
tests only monkeypatched the original 4 backends. The real Groq
backend was answering before the mocks could fire.

Fix: every test now monkeypatches `_try_groq` too, and the
expected order assertions now reflect Groq-first balanced
(`groq → gemini → gemma → anthropic → ollama`) and fast
(`groq → gemini → gemma → ollama → anthropic`), with premium
unchanged at the head (`anthropic → gemini → groq → gemma →
ollama`).

After the fix: 11/11 llm_tiers tests pass.
