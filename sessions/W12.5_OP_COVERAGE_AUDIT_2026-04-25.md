## Bugs surfaced by the audit (pre-run)

### Bug 1 ‚Äî `coil` naming mismatch ‚Äî FIXED 2026-04-25
- Few-shot `helical_spring.json` emits `kind: "coil"`.
- Rhino bridge dispatched `helix` (guide curve only ‚Äî no solid).
- Onshape executor implemented `_op_helix` only.
- **Fix applied**: `OpCoil` added to Rhino (`AriaBridge.cs`) ‚Äî builds
  spiral path, transforms section sketch onto perpendicular frame at
  spiral start, sweeps to solid. `_op_coil` added to Onshape executor ‚Äî
  posts helix feature + sweep feature with helix-fid as path query.
  Plug-in rebuild required for Rhino to see it.

### Bug 2 ‚Äî Rhino missing `gearFeature`
- `spur_gear.json` emits `gearFeature` once.
- Onshape and Fusion both have it; Rhino throws "Unknown feature kind".
- **Fix**: implement `OpGearFeature` using involute tooth geometry (mirror
  of `_cq_involute_gear` in cadquery_generator.py).

### Bug 3 ‚Äî Implicit (SDF) ops not bridged on Rhino + Onshape
- `gyroid_bracket.json` uses `implicitInfill`.
- `conformal_cooling.json` uses `implicitChannel`.
- Both currently throw on Rhino + Onshape.
- **Fix**: planner should emit a precursor `meshImportAndCombine` op that
  references a pre-built SDF STL. The implicit ops are a Fusion-only
  shortcut; the architectural answer is that the SDF backend produces
  the STL externally and CAD hosts just import it. Re-shape the two
  affected few-shots accordingly.

## Blocked on user

1. Close Rhino ‚Üí rebuild plug-in via `.\build.ps1` from
   `cad-plugins\rhino\AriaPanel\` ‚Üí reopen Rhino ‚Üí confirm
   `AriaGenerate` opens the panel and loads the React frontend.
2. Provide Onshape DID/WID/EID env vars
   (`ARIA_ONSHAPE_INTEGRATION_DID/WID/EID`) for the dedicated integration
   Part Studio.
3. Once both available: `python scripts/integration_test.py` will run
   Rhino + Onshape unattended and produce the cross-host report.
