# W1-W10 test audit — rigor pass

## What I did

Inventoried every public API across the W1-W10 modules
(`def <fn>(`/`class <Cls>:` not prefixed with `_`), grep-counted
how many test files reference each symbol, and built a coverage
matrix.

## What the audit found

### 22 public APIs with **zero** test references

The most damaging gap was the **W8 multi-modal entry points**. Every
agent has a public `*_to_plan(...)` function — sketch_to_plan,
image_to_plan, multiview_to_plan, scan_to_plan, voice_to_plan —
and not one was being called by any test. Only the helpers
(intent classifiers, JSON parsers, target resolvers) were
covered. So the path "user invokes the agent → vision/STT call →
goal extraction → dispatcher → plan returned" was untested.

Other untested public APIs:
- W3 topology optimization: `optimize_cantilever`,
  `density_to_sdf` (entire module)
- aria_os.feedback: `feedback_dir` helper
- aria_os.verification: `VerifyReport` dataclass (only
  reachable via `verify_part`, but its `to_dict()` shape was
  never asserted)
- scripts/ab_eval.py: `_compare()` function (the comparison
  logic that drives the W11 CI gate)
- 12 inherited engineering helpers (clearance_hole_mm,
  parse_bolt_spec, surface_finish_default, etc.) — pre-existing
  W6 setup; left as-is since they were working before W1-W10

### Real bug found by writing the missing tests

`scripts/promote_fewshots.py:write_fewshot_files` did NOT write
the `plan_hash` field into the auto-promoted few-shot JSON files.
But `prune_stale_auto_shots` reads `d.get("plan_hash")` to decide
which files to keep — so every auto-promoted file would get
pruned on the very next run.

This is a silent data-loss bug: auto-promotion would appear to
work (writes the file), then on the next nightly run, every
single auto-promoted file would be pruned because none of them
had the `plan_hash` the keeper needed.

**Fix:** added `plan_hash: ph` to the payload dict + a comment
explaining why. Test `test_write_files_creates_auto_prefixed`
now asserts `data["plan_hash"] == "abc123"` so the regression
can't come back.

## What I added

### test_multimodal.py::TestW8EntryPoints (11 new tests)

Closes the W8 entry-point gap. Each agent test:
1. Stubs the vision LLM / STT helper (no API credit burned)
2. Stubs the dispatcher's `make_plan` (deterministic plan
   shape, with `_captured_goal` smuggled into the last op so
   the test verifies the goal flowed through)
3. Asserts the full return shape (goal, spec, plan, modality-
   specific fields like part_family / confidence /
   resolved_target / analysis)
4. Negative paths: missing file raises FileNotFoundError,
   unavailable backend raises RuntimeError, too-few/too-many
   views raise ValueError

### tests/test_audit_gaps.py (19 new tests)

- `TestTopoptCantilever` (4 tests): shape, volume fraction
  respected, fixed face kept solid (load can react),
  density_to_sdf returns a callable that produces array output.
- `TestPromoterFileSystem` (3 tests): auto_-prefixed file
  writing with **plan_hash field assertion** (the bug-catcher),
  dry-run writes nothing, prune skips curated files + only
  removes stale.
- `TestAbCompare` (3 tests): no-diff comparison, regression
  detection (PASS→FAIL flagged), improvement detection
  (FAIL→PASS).
- `TestFeedbackEdgeCases` (5 tests): feedback_dir creates
  missing dir, INDEX.jsonl appends in order, load_all skips
  malformed JSON, plan_hash on empty plan, plan_hash skips
  non-dict entries.
- `TestVerifyReport` (1 test): to_dict() serializes + JSON-
  roundtrips with Issue → dict conversion.

## Coverage gaps acknowledged but not addressed

- 12 pre-existing engineering helpers (W6 setup) without
  tests. They were inherited working from before W1-W10. Not
  dangerous; can fold into a future "engineering library
  rigor pass" if needed.
- Live LLM eval still blocked on free-tier daily quota
  saturation — same as the prior baseline. Code path is
  proven; what's missing is the empirical pass-rate measurement.
- Live integration tests for the host CAD plugin handlers
  (Fusion Python, Rhino C#, Onshape REST) — these can only
  be exercised against real CAD instances. Static coverage
  tests already pin "every W1 op kind has a corresponding
  Fusion / Onshape handler key", which catches the most
  common drift.
