### Impeller CadQuery template added
- `_cq_impeller()` added to `aria_os/generators/cadquery_generator.py`
- Open-face design: thin base disc (20% of height = 4mm) + blades fully exposed above (80% = 16mm)
- BT (blade thickness) = max(user_bt, OD * 0.10) = 15mm for 150mm OD — wide enough to render visibly
- Backward blade angle: each blade tip rotated -30 deg from its root angle
- Registered in `_CQ_PART_MAP`, `_CQ_TEMPLATE_MAP`, `_KEYWORD_TO_TEMPLATE`

### Wireframe renderer fixed (critical)
- Root cause: trimesh `scene.save_image()` fails headlessly on Windows (no GL context)
- Old fallback used matplotlib scatter plot — thin blades rendered as invisible dots
- Fix: use `mesh.edges_unique` to get all physical edges, draw as `LineCollection` (alpha=0.5, lw=0.4)
- Added convex hull border as solid blue line for orientation
- Subsample to 20,000 edges for dense meshes to avoid memory issues

## What Was Built

### Terrain generator (`aria_os/generators/terrain_generator.py`, 617 lines)

New synthetic terrain generation module — no external geometry engines, numpy-only heightmap.

- **Diamond-Square fractal heightmap**: grid must be 2^n+1 (257, 513, etc.). Roughness H=0.6 produces natural-looking mountains. Peak shaping applies radial distance weighting from grid centre.
- **Marching squares contour extraction**: vectorised numpy bitmask (16-case). ~6000 polylines from 257x257 mountain grid. Contours bucketed into minor (GRADE-EXIST-CONTOUR) and major/index (GRADE-EXIST-INDEX) layers.
- **Binary STL output**: struct.pack, 131,072 faces for 257x257 grid, ~6400 KB.
- **DXF output via ezdxf**: units=6 (metres), real-world scale. Two layers per civil/survey convention.
- **Terrain types supported**: mountain, hills, plateau, valley, ridge, flat.
- **`parse_terrain_params(description)`**: regex NL parser for size, peak elevation, terrain type.
- **`generate_terrain(description, output_dir)`**: top-level entry point, returns result dict.
- **Tested**: mountain 3km x 3km 150m peak → 6804 KB DXF, 6400 KB STL — pass.
