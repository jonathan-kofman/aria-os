### Changes Made

#### `aria_os/tool_router.py`
- All `return "fusion"` changed to `return "fusion360"` throughout the file.
- Root cause: orchestrator `_VALID_TOOLS` check compares against the string `"fusion360"`, not `"fusion"`. The mismatch caused every Fusion 360 routing decision to fail silently, falling through to a wrong tool or no-op.

#### `aria_os/tool_router.py` + `aria_os/multi_cad_router.py`
- Sheet metal goals removed from `FUSION_KEYWORDS`. Feature type `"sheet_metal"` now routes to CadQuery instead of Fusion 360.
- `multi_cad_router.py`: `_build_rationale` updated — `"fusion"` string check corrected to `"fusion360"`. Docstring also updated.

#### `aria_os/__init__.py`
- Changed to lazy imports. All `from aria_os.orchestrator import ...` and similar at module load time removed/deferred.
- Prevents Railway ImportError chain: if any optional dependency (grasshopper_generator, blender_generator, rhino3dm) is absent, the entire package was previously crashing on import. Now it degrades gracefully.

#### `aria_os/spec_extractor.py`
- Added gusset leg extraction: `"Nmm leg"` pattern → `leg_a_mm`, `leg_b_mm`.
- Box notation (`WxHxDmm`) now also maps `width_mm`/`depth_mm` correctly for gusset templates.

#### `aria_os/generators/cadquery_generator.py`
- **Gusset template**: legs now fall back to `width_mm`/`depth_mm` if `leg_a_mm`/`leg_b_mm` not present.
- **Weld bead workplane fix**: changed `cq.Workplane("YZ")` to `cq.Workplane("XY")` for polyline extrusion. The YZ workplane was producing ~484-byte degenerate STL (empty/zero-volume geometry).
- **U-channel**: added `_cq_u_channel` wrapper with `n_bends=2` default. Previously the u-channel keyword was routing to a single-flanged panel template.
- **Keyword routing**: `u_channel` keyword entry updated to point to new wrapper.

#### `dashboard/_build_argv`
- Added 18+ missing command→flag mappings: terrain, scan, catalog-search, assembly/assemble split, and others.
- Previously these dashboard commands had no argv translation, causing silent no-ops when invoked from the UI.

### Key Decisions

- Sheet metal stays on CadQuery (not Fusion 360) — Fusion 360 path is not available on Railway and has no headless runner. CQ sheet metal templates (gusset, u-channel, weld_bead) are sufficient for the current part catalog.
- Lazy imports in `__init__.py` are the correct long-term pattern for this package, not just a Railway workaround. Any package mixing optional heavy deps should never import them at module level.
- The `"fusion"` vs `"fusion360"` bug was silent for a long time because the fallback chain catches the routing failure without logging it. Added explicit log line.
