## 2026-04-06 — CQ template verification methodology
Test templates by running with trimesh post-check: watertight, component count, volume vs expected solid.
Volume alone isn't enough — need component count to catch floating disconnected bodies.
Execution test pattern that catches real bugs: solid_count check + watertight + volume_removal check.

## 2026-04-06 — Implied component dependencies in ECAD
When a motor driver (L298N) is detected, the board ALWAYS needs:
- 12V barrel jack (motor supply input)
- AMS1117-3.3 LDO (MCU logic supply)
These should be auto-added in parse_components() when not already present.
Same pattern: STM32/ESP32 always needs LDO regardless of whether "12V" appears in description.

## 2026-04-06 — CadQuery shell() for sheet metal boxes
cq.Workplane("XY").box(L,W,H).faces(">Z").shell(-T) creates an open-top tray.
Negative shell value = inward, removing the selected face. Clean single solid, watertight.

## 2026-04-07 — KiCad 10 headless rendering: use kicad-cli, never pcbnew Python for rendering
pcbnew Python scripts generated by ARIA-OS are GUI console scripts — they cannot be exec()'d headlessly.
kicad-cli is the correct tool for automated rendering:
  `kicad-cli pcb export svg <file.kicad_pcb> --layers F.Cu,B.Cu,F.SilkS,Edge.Cuts`
To generate a .kicad_pcb for rendering: write KiCad 10 S-expression format directly from BOM data.
KiCad 10 format reference: version=20241229, gr_line for board outlines, fp_line for footprint body,
all elements require uuid. This approach is stable, reproducible, and bypasses all headless API issues.

## 2026-04-07 — KiCad Python can be imported headlessly for simple operations only
Works headlessly: `import pcbnew`, `pcbnew.Version()`, `pcbnew.BOARD()`, `pcbnew.FromMM()`, `pcbnew.VECTOR2I()`
Requires: run with cwd=KiCad bin dir, use relative "Lib/site-packages" in sys.path (not absolute path).
Crashes on: any complex board manipulation — PCB_TEXT, FOOTPRINT construction, add/render operations.
Rule: use pcbnew Python only for unit/value conversions, never for board construction in automated pipelines.

## 2026-04-07 — Output verifier architecture in aria_os/output_verifier.py
Three verification domains, each with geometry checks + optional render:
- verify_ecad(): 9 placement/connectivity checks + KiCad board render (kicad-cli SVG → matplotlib fallback)
- verify_dxf(): 8 geometric checks + ezdxf matplotlib render
- verify_stl(): 7 mesh health checks + 3-view trimesh render
Key helpers: _bom_to_kicad_pcb() converts BOM JSON → .kicad_pcb; render_kicad_board() manages kicad-cli subprocess.
CLI access: `python run_aria_os.py --verify <file>` or `python -m aria_os.output_verifier --ecad/--dxf/--stl`

## 2026-04-07 — cairosvg exception handling on Windows
cairosvg failure on Windows is an OSError (missing DLL), not ImportError.
Always wrap cairosvg import and calls in `except Exception` not `except ImportError`.
Matplotlib is the reliable fallback for SVG/PNG rendering on Windows without GTK stack installed.

## 2026-04-07 — Always use .is_file() not .exists() when checking CAD output paths
- .exists() returns True for directories and "." (empty path)
- .is_file() is the correct check for ensuring actual output files were produced
- Applies to: orchestrator.py, any code that checks step_path/stl_path before processing

## 2026-04-07 — Agent mode recovery pattern
- Agent stall (no LLM available) should not be a hard failure — templates can always produce geometry
- Pattern: check if geometry was produced after agent loop; if not, reset _use_agents=False to fall through to legacy path
- session.pop("checkpoints") before fallback so legacy path starts fresh

## 2026-04-07 — Dashboard live stream fix
- `selectRun()` was closing the SSE stream after 3 seconds (designed for replay). When a user clicks a RUNNING run in the left panel, the stream closed before the run finished. Fix: extracted `_attachStream(id, isLive)` helper — `isLive=true` keeps the stream open until done; `isLive=false` closes after 8s for replay.
- `init()` with `?from_vis=<run_id>` now calls `_attachStream` instead of inline EventSource logic — all stream management goes through one place.
- `startRun()` also now delegates to `_attachStream`.
- Run history persistence: `_save_run_log()` writes completed runs (last 200 lines each) to `outputs/dashboard_run_log.json` after every run. `_load_run_log()` reloads on startup — solves the problem where server restarts lose run history and `?from_vis` links break.
- Three.js STL preview: use importmap + `<script type="module">` to expose `window.THREE`, `window.STLLoader`, `window.OrbitControls`. Fetch STL via `/api/file?path=` (security-checked), parse with `STLLoader.parse(arrayBuffer)`, center geometry on origin before rendering.
- StructSight site image passthrough: stored as `site_image_b64` + `site_image_type` in run record; shown as thumbnail in the pipeline banner when `?from_vis` param is present.
